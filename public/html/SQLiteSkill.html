基本概念

  • 資料庫 (Database): 用來存放資料的容器，每個資料庫都包含多個表。
  • 表格 (Table): 資料以表格的形式來儲存，由行和列組成。
  • 行 (Row): 表格中的每條記錄。
  • 列 (Column): 表格中的每個屬性，描述資料的每個特徵。
  • 主鍵 (Primary Key): 唯一識別表中每一條記錄的欄位。
  • 外鍵 (Foreign Key): 用來建立和維護表之間的關聯的欄位。外鍵的值對應於另一個表的主鍵，用來確保數據的一致性和完整性。

資料類型概述

  • 整數類型: INTEGER
  • 字串類型: TEXT
  • 日期時間類型: DATE、DATETIME
  • 浮點數類型: REAL

創建使用資料庫

在 SQLite 中，選擇或創建資料庫時，直接在命令行中打開資料庫文件：sqlite3 my_database.db

新增表格

使用 CREATE TABLE 來新增表格：

CREATE TABLE table_name (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  column1 TEXT NOT NULL,
  column2 INTEGER
);
  • INTEGER: 整數類型
  • AUTO_INCREMENT: 自動增加欄位值
  • NOT NULL: 列不能空，所以它不能包含 NULL 值
  • TEXT:不固定長度字串

SELECT 語句

使用 SELECT 來查詢表中的資料：

SELECT * FROM table_name;

INSERT INTO

使用 INSERT INTO 來插入數據：

INSERT INTO table_name (column1, column2) VALUES (value1, value2);

UPDATE 語句

使用 UPDATE 來更新表中的資料：

UPDATE table_name 
SET column1 = value1, column2 = value2 
WHERE condition;

DELETE 語句

使用 DELETE 來刪除資料：

DELETE FROM table_name 
WHERE condition;

JOIN 語句

使用 JOIN 來聯接兩個或多個表：

SELECT columns 
FROM table1 
JOIN table2 ON table1.common_field = table2.common_field;

改變表格

使用 ALTER TABLE 來修改表格結構：

ALTER TABLE table_name 
ADD column_name datatype;

移除表格

使用 DROP TABLE 來刪除表格：

DROP TABLE table_name;

查詢條件與過濾

  • 條件查詢: WHERE子句用於根據條件過濾查詢結果：
    • SELECT * FROM users WHERE name = 'John Doe';
  • 多條件查詢: 使用 AND 和 OR 結合多個條件：
    • SELECT * FROM users WHERE name = 'John Doe' AND email LIKE '%example.com';
  • 範圍查詢: 使用 BETWEEN 和 IN：
    • 
SELECT * FROM users WHERE id BETWEEN 1 AND 10; 
SELECT * FROM users WHERE id IN (1, 2, 3);
  • 模糊查詢: 使用 LIKE：
    • SELECT * FROM users WHERE email LIKE '%example.com';

聚合函數

聚合函數用於計算集合數據的統計信息：

  • COUNT: 計算行數
    • SELECT COUNT(*) FROM users;
  • SUM: 計算總和
    • SELECT SUM(amount) FROM orders;
  • AVG: 計算平均值
    • SELECT AVG(amount) FROM orders;
  • MAX: 查找最大值
    • SELECT MAX(amount) FROM orders;
  • MIN: 查找最小值
    • SELECT MIN(amount) FROM orders;

GROUP BY

GROUP BY 用於分組查詢

SELECT column_name, COUNT(*) 
FROM table_name 
GROUP BY column_name;

HAVING 子句

HAVING 用於過濾分組後的結果

SELECT column_name, COUNT(*) 
FROM table_name 
GROUP BY column_name 
HAVING COUNT(*) > 1;

ORDER BY

ORDER BY 用於對查詢結果進行排序

SELECT * FROM table_name 
ORDER BY column_name ASC/DESC;

LIMIT

LIMIT 用於限制查詢結果的行數

SELECT * FROM table_name 
LIMIT 10;

子查詢

子查詢是嵌套在其他查詢中的查詢：

SELECT * FROM table_name 
WHERE column_name = (SELECT MAX(column_name) FROM another_table);

視圖

  • 創建視圖: 保存查詢結果以供未來使用
    • CREATE VIEW view_name AS 
SELECT column_name 
FROM table_name;
  • 刪除視圖: 移除視圖
    • DROP VIEW IF EXISTS view_name;

索引

    索引是提高查詢效率的重要工具。它能夠加速資料庫的檢索操作。以下是常見的索引操作及其句型和填入部分：
  • 創建索引: 使用 CREATE INDEX 語句來創建索引，並指定表和索引列。
    • 句型: 
      CREATE INDEX index_name ON table_name (column_name);
    • 填入部分: index_name、table_name、column_name
      • index_name: 索引的名稱。
      • table_name: 要添加索引的表名。
      • column_name: 表中的列名，該列將被索引。
  • 刪除索引: 使用 DROP INDEX 語句來刪除索引。33
    • 句型: 
      DROP INDEX index_name ON table_name;
    • 填入部分: index_name、table_name
      • index_name: 要刪除的索引名稱。
      • table_name: 索引所在的表名。
  • 查看索引: 使用 SHOW INDEX 語句來查看表中的索引。
    • 句型: 
      SHOW INDEX FROM table_name;
    • 填入部分: table_name
      • table_name: 要查看索引的表名。
  • 添加唯一索引: 使用 CREATE UNIQUE INDEX 語句來創建唯一索引，確保列中的值唯一。
    • 句型: 
      CREATE UNIQUE INDEX index_name ON table_name (column_name);
    • 填入部分: index_name、table_name、column_name
      • index_name: 索引的名稱。
      • table_name: 要添加索引的表名。
      • column_name: 表中的列名，該列將被索引，且值必須唯一。

事務

    事務（Transaction）用於確保一組操作要麼全部成功，要麼全部失敗，以保持數據的一致性。事務包括開始、提交和轉返操作。
  • 開始事務: 使用 BEGIN TRANSACTION; 開始事務。
  • 提交事務: 使用 COMMIT; 來提交事務。
  • 回滾事務: 使用 ROLLBACK; 回滾事務。

注意事項：

  • 事務應該被用來包裹一組互相關聯的操作，以確保數據的完整性。
  • 如果在事務中發生錯誤，應使用 ROLLBACK 撤銷操作，並重新嘗試或處理錯誤。
  • 每個事務的結束都必須有 COMMIT 或 ROLLBACK，否則事務會保持打開狀態，可能會影響系統性能。

觸發器

    觸發器（Trigger）是與表操作（如插入、更新、刪除）相關聯的自動執行的 SQL 語句集合。
  • 創建觸發器: 當表發生特定事件時執行動作 
    CREATE TRIGGER trigger_name 
AFTER INSERT ON table_name 
BEGIN UPDATE another_table 
SET column_name = value; 
END;
  • 刪除觸發器: 使用 DROP TRIGGER IF EXISTS trigger_name; 來移除觸發器。

注意事項：

  • 觸發器可以自動執行特定的操作，例如審核數據變更或同步表。
  • 注意觸發器的執行時機和操作類型，以確保正確性。
  • 過多的觸發器可能會影響性能，應謹慎使用。

性能調優

  • 使用索引: 創建索引以加快查詢速度。 
    CREATE INDEX index_name ON table_name (column_name);
  • 避免不必要的 DISTINCT 和 ORDER BY 來提高性能。

資料庫遷移和升級

  • 手動遷移: 可以使用.dump導出數據到 SQL 文件，然後在新的數據庫上導入。
  • 升級: 更新到新版本的 SQLite 會自動處理數據庫升級。
